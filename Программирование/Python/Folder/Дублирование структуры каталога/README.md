# Дублирование структуры каталогов

## Описание
Предназначен для копирования (дублирования) структуры папок (подкаталогов) из указанного исходного каталога в целевой каталог, без копирования самих файлов.

## Назначение
- Создание точной иерархии папок исходной директории в новом месте.
- Подготовка среды или шаблона папок для новых проектов.
- Резервное копирование структуры каталогов.

## Возможности
- Рекурсивный обход всех подкаталогов в исходной директории.
- Создание целевой структуры с сохранением относительных путей.
- Автоматическая проверка длины пути (ограничение **255 символов**), что полезно для совместимости с ограничениями файловых систем, например, в Windows.
- Генерация отчета (`info.txt`) в целевом каталоге, который фиксирует:
    - Полные пути пропущенных каталогов (из-за превышения лимита длины пути).
    - Ошибки при создании каталогов.
- Простой и интуитивно понятный интерфейс командной строки для ввода путей.
- Создание целевого каталога, если он не существует.

## Принцип работы

1.  **Запуск и ввод данных**: Пользователь запускает скрипт и вводит абсолютный или относительный путь к исходному каталогу и целевому каталогу.
2.  **Проверка**: Скрипт проверяет существование исходного каталога и при необходимости создает целевой.
3.  **Сбор подкаталогов (`collect_directories`):**
    * Используется функция `os.walk()` для рекурсивного обхода исходного каталога.
    * Собираются полные пути всех найденных подкаталогов.
4.  **Создание структуры (`create_target_structure`):**
    * Для каждого найденного каталога вычисляется относительный путь относительно исходного каталога (`os.path.relpath`).
    * Формируется полный путь нового каталога в целевой директории.
    * **Проверка длины:** Если полный путь превышает **255 символов**, путь записывается в файл `info.txt` и каталог пропускается.
    * **Создание:** Используется `os.makedirs(..., exist_ok=True)` для создания каталога и всех необходимых родительских папок.
    * В случае ошибки создания (например, проблемы с правами доступа), ошибка записывается в `info.txt`.
5.  **Отчет:** По завершении работы выводится сводка с количеством созданных и пропущенных каталогов.
